什么是 AI Agent？
它最适合用来做什么？
如果我们想开始自己动手做，又需要准备哪些东西？

这支视频就是 AI Agents for Beginners 课程的第一课，我们会先把这些基础问题讲清楚。
整个课程会带你从概念走到代码，理解怎样一步步把 AI Agent 真正做出来。
这支短视频对应配套的文字课程，里面还有翻译和示例代码，链接就在视频上下方。

先来看一个 AI Agent 最核心的几个组成部分。
第一部分是大语言模型，也就是驱动 Agent 进行推理的核心。
这里说的推理，不是一个很抽象的词，而是指它能识别用户要完成什么任务，先想出一个计划，再一步步执行这个计划。

第二部分是记忆。
记忆可以分成短期记忆，也就是当前这段对话里的上下文；
也可以是长期记忆，也就是一组能让 Agent 随着时间不断改进任务表现的数据。

第三部分是工具。
工具可以是通过 API 调用的服务，可以是帮助判断下一步动作的数据源，也可以是我们主动接给 Agent 的函数。

把这些东西组合起来以后，Agent 会利用大语言模型理解用户目标，判断需要调用哪些工具，再结合记忆里已有的信息，完成用户交给它的任务。

视频里举了一个很生活化的例子，就是刷牙。
你得先知道什么时候刷、在哪里刷；
你的工具是牙刷和牙膏；
你有当前情境里的记忆，比如现在是不是已经刷到一半；
你也有长期记忆，比如你更喜欢哪一种牙膏口味。

当然，我们今天不是来研究刷牙的，我们是来动手做 AI Agent 的。
所以接下来，镜头会切到代码编辑器。

在这个示例里，课程作者用的是 Microsoft 的 Semantic Kernel，以及 GitHub Models 提供的免费大语言模型接口。
如果你是第一次接触 Jupyter Notebook，或者还不知道怎么把环境搭起来，课程里其实也有专门的章节讲这部分。

这一课不会把 Semantic Kernel 的细节展开得太深，因为后面的章节会专门比较不同的 Agent Framework，看看它们分别是怎么工作的，以及各自的优缺点。

不过这里有个地方很值得注意，就是这个 destinations plugin。
你可以把它理解成一组提前定义好的工具函数，供 Agent 在需要的时候调用。

在这个例子里，Agent 手上有一份世界各地旅行目的地的列表。
同时还有一个函数，每调用一次，就会随机从这个列表里挑出一个城市返回给用户。

这就是为什么前面讲 Agent 时，我们一直强调工具的重要性。
我们不是让模型凭空想象，而是给它一组明确可用的能力。

然后，这个示例还会把工具挂到 Agent 上，同时给 Agent 指定一个大语言模型。
视频里用的是 GPT-4o mini。
你也完全可以自己换成别的模型，看看它们的表现有什么不同。

接下来作者模拟了一段用户和 Agent 的真实交互。
用户先说：帮我计划一次一日游。
这里很能体现大语言模型的优势，因为用户并没有严格按照固定命令输入，而是用自然语言表达需求。
模型需要自己理解：用户是在请求一次旅行建议，而且因为没有指定城市，所以可能需要调用那个随机目的地函数。

然后，示例里还顺便测试了记忆能力。
如果用户不喜欢第一次推荐的地方，再说一句：换一个目的地，重新帮我规划一次。
这时候，如果 Agent 真的保留了对话上下文，它就不应该重复刚才那个答案。

运行结果里，用户第一次得到的是悉尼。
当用户表示不喜欢之后，Agent 又重新给出了里约热内卢这个目的地。
这说明它一方面确实调用了我们提供的工具，而不是随便从自己知道的城市里乱选；
另一方面，它也利用了当前对话的上下文，知道应该换一个新的选项，而不是重复上一次的结果。

所以，这个小例子虽然简单，但已经把 Agent 的几个关键点串起来了：
大语言模型负责理解意图，工具负责提供可执行能力，记忆负责保留上下文。

这就是第一课的核心内容。
下一课会继续往下讲，带你看看现在常见的几种 Agent Framework 到底有什么区别。
