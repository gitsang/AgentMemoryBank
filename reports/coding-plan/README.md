#### 计算说明

**价值估算** = 月订阅价 × 15（低） ∼ × 30（高），即官方宣称的 "15-30 倍 API 等价额度"。

**Token 有效单价**（80% 缓存命中率，100:1 输入输出比例）：

```
智谱: (0.2×¥8 + 0.8×¥2) × 100 + ¥28 × 1  /  101 = ¥3.45 / M token
Z.AI: (0.2×$1.4 + 0.8×$0.26) × 100 + $4.4 × 1  /  101 = $0.527 / M token
```

Tokens 估算 = 价值（低 ∼ 高）÷ 有效单价。

**惩罚率** = `1 − 1/(高峰比例 × 3 + 低峰比例 × 倍率)`。两个值分别对应低峰 1×（限时福利）和低峰 2×（正式费率）。惩罚率只取决于模型系数和高峰占比，与套餐级别无关。

GLM-5.2 高峰 3× 消耗、低峰 1×（限时福利至 9 月底，之后恢复 2×）。高峰时段：14:00-18:00 (UTC+8)。

#### 结论

- Z.AI 套餐虽然订阅价高（≈3× 国内），但单位 token 成本更低（¥3.45 vs $0.527 ≈ ¥3.79），同等订阅价买到更多 token。
- 惩罚率在 50% 高峰时达 50%-60%，意味着 GLM-5.2 实际可用 prompt 数约为官方配额的一半。高峰越多，惩罚越重。
- 你的 6K/周 + GLM-5.2 场景即使 Max 也装不下，直接 API 按量更划算。

### 总结

整体来说是非常不划算的。

对于高强度使用的情况，每天应该消耗在 0.5-2 亿 Token，这个套餐算总配额也就只够用一天。就算是低强度，大概也就只够用一周左右。

但这也是为数不多国内能够用上 GLM-5.2 的性价比比较高的途径（官方的套餐基本上可以认为是不放货的，只能买海外版贵三倍）。

比较适合想要尝鲜试用国内模型的用户。

>
