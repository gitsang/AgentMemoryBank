好，现在先来看看 AI Agent 的组成部分。
第一部分是大语言模型，
它负责 Agent 背后的推理。
这里说的推理，就是它能识别
用户提出的任务，制定完成计划，
再把计划里的动作执行出来。
下一部分是记忆。
它可以是短期记忆，也就是
用户和 Agent 当前对话的上下文；或者是长期记忆，
也就是一组数据，让 Agent
随着时间更擅长完成任务。
再接下来是工具。
工具可以是通过 API 调用的服务，
可以是真正执行动作的能力，也可以是帮助判断
下一步该做什么的数据，或者各种函数，
我们再把这些信息发给 AI Agent。
当这些部分组合起来以后，
Agent 会利用大语言模型识别
用户到底想完成什么任务，
再判断需要哪些可用工具
来完成这个任务，并且用记忆来收集信息，
以及完成任务所需的数据。
一个简单的现实例子，而且你最好每天都在做，
就是刷牙。
你得先计划什么时候刷、在哪里刷。
工具就是牙刷和牙膏。
你还有一些上下文记忆，
知道自己现在刷到什么状态了，
也有长期记忆，
比如你更喜欢哪种牙膏口味。
我自己更喜欢留兰香。
谢谢你关心。

我们会稍微模拟一下

我觉得这个例子很好地说明了
使用大语言模型的一种能力，
就是用户可以直接用自然语言沟通，
比如他不会明确说“我要度假”，
只是说想来个一日游，而大语言模型
应该能够理解这其实是在请求
或者至少需要调用这个函数，
也就是 random destinations，因为
用户并没有提供具体城市。
而且大语言模型也知道
自己被分配了哪些工具。
接着我们还会顺便演示
记忆本身的一点作用，

然后让 Agent 再规划一次。
如果 Agent 保留了上下文，
也就是这段对话里的记忆，
它就不应该重复刚才的目的地。
