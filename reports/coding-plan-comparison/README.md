# AI Coding/Token Plan 定价对比（2026年7月）

> 数据采集日期：2026-07-01 | 汇率参考：1 USD ≈ 7.2 CNY

---

## 换算说明

### Prompt → Token 换算

- **GLM Coding Plan**: 官方声明月可用额度按 API 定价折算 = 月订阅费 × 15~30 倍。按典型 Coding 场景（70% 输入 + 30% 输出，缓存比例约 80% 输入缓存命中）估算。
- **Kimi Coding Plan**: Token 配额制，按套餐实际标注的 Token 限额。
- **MiniMax Token Plan**: 官方标注 ~X B tokens/month of M3 usage。直接采用官方数据。
- **Mimo Token Plan**: Credit 制，按 mimo-v2.5-pro 的 Credit/token 消耗比例换算（假设缓存命中:未命中:输出 = 60%:10%:30%）。
- **阿里云 Token Plan**: Credit 制，按 qwen3.7-max API 定价反算 Credits 对应的 Token 量。
- **Opencode Go**: 美元配额制，$60/月上限，按各模型 API 价格反算 Token 量。

### 月配额美元换算

- 订阅计划类：直接使用月订阅价格（美元）。
- API 按量付费类：无固定月配额，标注 "按量"。
- Opencode Go：$10/月订阅费，获得 $60/月 API 等值额度。

### 峰值标注

- GLM-5.2/GLM-5-Turbo：高峰期（14:00~18:00 UTC+8）3× 消耗，非高峰期 2×（至 2026年9月底限时 1×）。
- Kimi：无高峰期系数。
- MiniMax M3：Priority 模式 1.5× 标准价。
- 其他：无特殊高峰期标注。

### GLM-5.2 月配额换算说明

**Z.AI Coding Plan Token 换算**：

- 官方声明：月可用额度 ≈ 月订阅费 × 15~30 倍（按 API 定价折算）
- Lite $18 × 15~30 = $270~$540 API 等值
- 按 GLM-5.2 API 价 ($1.40 输入 + $4.40 输出)，典型 Coding 混合价 ~$2.30/1M tokens
- Token 估算：$270 ÷ $2.30 × 1M ≈ 117M（下限），$540 ÷ $2.30 × 1M ≈ 235M（上限）
- 实际因缓存（$0.26/M cache read），有效 Token 量远高于此估算

**Opencode Go Token 换算**：

- $60/月 ÷ ($1.40 输入 + $4.40 输出) 混合 ≈ $60 ÷ $2.30 × 1M ≈ 26M tokens（无缓存）
- 实际因缓存可大幅提升

### Mimo-V2.5-Pro 月配额换算说明

**Credit → Token 换算**（以 Lite 套餐 4.1B Credits 为例）：

- 假设 Coding 场景：60% 缓存命中输入 + 10% 缓存未命中输入 + 30% 输出
- 每 1M tokens 消耗 = 600K × 2.5 + 100K × 300 + 300K × 600 = 1.5M + 30M + 180M = 211.5M Credits
- Lite 可用 Tokens ≈ 4.1B ÷ 211.5 × 1M ≈ 19.4M tokens/月
- 若全为缓存命中输入：4.1B ÷ 2.5 × 1M ≈ 1,640B tokens → 不合理（Credit 扣减比例限制了纯缓存场景）

**实际等效 Token 量**取决于输出比例。输出 token 消耗 Credits 最高（600 Credits/token），是缓存命中的 240 倍。
